# DB Driver Approval Target Binding

DB driver approval is bound to a precise target. It is not a general permission
to add database dependencies or run live DB tests.

## Binding Fields

- `target_commit_sha`: exact PR head SHA approved by the project owner.
- `base_commit_sha`: expected base SHA when the approval was created.
- `target_branch`: exact branch name.
- `pr_number`: exact pull request number.
- `repository`: `hiro4649/CRIPTO-TIP`.

## Why Regeneration Is Required

If code changes after approval, the target commit changes. The old approval no
longer covers the new code and must be regenerated by a project owner.

PR body edits may require evidence refresh. They do not create owner approval,
and they do not permit scope expansion unless the owner approval record is
regenerated for the new target commit and scope.

## Current Boundary

This PR does not approve a DB driver, real DB connection, package change,
lockfile change, migration execution, provider SDK apply, or production
deployment.
