# Character Continuity Contract

CRIPTO-TIP does not run Live2D, TTS, VOXWEAVE orchestration, IRIS Core AI conversation, renderer execution, OBS delivery, or WebSocket delivery. CRIPTO-TIP creates safe constraints and safe support context that can be handed to the systems that own those responsibilities when future owner scope authorizes integration.

## Boundary

CRIPTO-TIP owns:

- support event normalization
- safe viewer identity
- safe context summary
- moderation and operator state
- reaction and overlay candidate contracts
- audit-safe metadata

IRIS Core owns AI reaction generation, character memory, affinity interpretation, and persona policy application. VOXWEAVE owns voice orchestration when authorized. LIVE2D owns rendering and motion execution. CRIPTO-TIP must not execute those systems in this contract.

## Continuity Rules

- `persona_version` is managed by the operator.
- `character_id` selects the character contract, not a viewer-provided identity.
- `voice_profile_id`, `motion_profile_id`, and `overlay_theme_id` are controlled configuration.
- Payment amount is not persona override authority.
- A high Tip cannot change persona, voice, motion, relationship boundary, or identity.
- Viewer display names and messages cannot override character settings.
- A viewer instruction such as changing the character role, owner, relationship, or private behavior must be blocked by safe context generation.

## Forbidden Outcomes

The continuity contract rejects:

- token price or investment return discussion
- wallet address reading
- romantic escalation from payment
- ownership or control rights from payment
- private or secret behavior promises from payment
- persona override through viewer name or message
- character identity changes caused by support amount

This document is a specification. It does not claim runtime readiness, production readiness, legal compliance, or YouTube policy compliance.
